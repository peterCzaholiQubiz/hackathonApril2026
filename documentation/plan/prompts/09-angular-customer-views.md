# Prompt 09 — Angular Customer List & Detail Views

**Agent**: `general-purpose`  
**Phase**: 3 — AI + Frontend  
**Status**: DONE  
**Depends on**: Prompt 08 (dashboard + shared components)

---

Implement the customer list, customer detail, and risk groups features for the
Customer Portfolio Thermometer.

Read documentation/plan.md (Frontend section).

CUSTOMER LIST — frontend/src/app/features/customer-list/

1. customer-list.component.ts + .html + .scss
   - Calls GET /api/customers with query params on init and filter change
   - Paginated table: HeatBadge | Name | Company | Segment | Churn | Payment | Margin | Overall
   - Score columns use ScoreBarComponent
   - Click row navigates to /customers/:id
   - Pagination controls (prev/next, page number, total count)

2. customer-filters/customer-filters.component.ts + .html + .scss
   - Segment dropdown: All / Enterprise / Mid-Market / SMB
   - Heat level toggle buttons: All / Green / Yellow / Red (visual toggle, not dropdown)
   - Sort by: Overall Score / Churn / Payment / Margin (dropdown)
   - Sort direction: ASC / DESC toggle
   - Name search: debounced input (300ms)
   - Emits filter change events to parent, does not call API directly

CUSTOMER DETAIL — frontend/src/app/features/customer-detail/

3. customer-detail.component.ts + .html + .scss
   - On init: fire 4 parallel API calls using forkJoin:
     GET /api/customers/{id}, /api/customers/{id}/risk,
     /api/customers/{id}/interactions, /api/customers/{id}/complaints
   - Header: HeatBadge, customer name, company, segment, account manager, onboarding date
   - Composes the four child components below

4. risk-breakdown/risk-breakdown.component.ts + .html + .scss
   - Three RiskGaugeComponents side by side: Churn Risk | Payment Risk | Margin Risk
   - Each gauge shows: dimension label, score/100, heat color arc

5. shared/components/risk-gauge/risk-gauge.component.ts + .scss (implement fully)
   - Input: label: string, score: number (0-100)
   - Semicircular SVG gauge, arc color based on score (green/yellow/red thresholds)
   - Score number centered below arc

6. explanation-panel/explanation-panel.component.ts + .html + .scss
   - Tab navigation: Churn | Payment | Margin | Overall
   - Each tab shows:
     - Score badge with heat color
     - AI-generated explanation text (labeled "AI Advisory" with info icon)
     - Confidence badge: High / Medium / Low
     - Info tooltip: "Risk scores are computed by deterministic rules.
       This explanation is AI-generated by Claude claude-sonnet-4-6 and is advisory only."
   - Loading skeleton while fetching

7. actions-panel/actions-panel.component.ts + .html + .scss
   - List of suggested action cards
   - Each card: priority badge (high=red, medium=amber, low=green) + action_type icon +
     title (bold) + description text
   - Action type icons: outreach=📞, discount=🏷, review=🔍, escalate=⚠, upsell=📈
   - Sort by priority (high first)
   - Empty state if no actions

8. customer-timeline/customer-timeline.component.ts + .html + .scss
   - Merged chronological list of interactions + complaints, most recent first
   - Interactions: channel icon + date + direction badge + summary text +
     sentiment chip (colored: positive=green, neutral=gray, negative=red)
   - Complaints: date + severity badge + category + description + resolved/unresolved status
   - Alternating left/right timeline layout

RISK GROUPS — frontend/src/app/features/risk-groups/

9. risk-groups.component.ts + .html + .scss
   - Three columns: Churn Risk | Payment Risk | Margin Risk
   - Each column lists top 10 customers by that dimension score, highest first
   - Each row: HeatBadge + customer name + score (ScoreBar)
   - Column header shows the avg score for that dimension
   - Click customer navigates to /customers/:id
